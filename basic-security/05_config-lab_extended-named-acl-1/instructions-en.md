# Extended Named ACL

## Objectives

Configure an extended named ACL to implement granular traffic filtering using source, destination, protocol, and port criteria.

## Instructions

1. Define the traffic filtering policy based on the network requirements.
2. Create an extended named ACL with a meaningful name.
3. Add permit and deny statements for the required traffic types.
4. Apply the ACL to the most efficient interface and direction.
5. Test and verify the ACL behavior.

## Verification

- Use `show access-lists` to verify the named ACL.
- Use `show ip interface` to verify ACL application.
- Test connectivity with appropriate applications and protocols.
